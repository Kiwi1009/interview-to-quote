# 簡易上傳介面使用說明

## 功能說明

這是一個簡化的中文介面，讓您可以：

1. **上傳會議逐字稿**（必填）
2. **上傳工廠機器照片**（可選，可多選）
3. **自動生成文件**：
   - 需求規格書（DOCX/PDF）
   - 需求報告表（DOCX/PDF）
   - 報價單（DOCX/PDF，包含三個方案）
4. **下載生成的文件**

## 使用步驟

### 1. 啟動服務

確保後端和前端服務都在運行：

```bash
# 終端1：後端
cd backend
python run.py

# 終端2：Celery Worker
cd backend
celery -A app.celery_app worker --loglevel=info

# 終端3：前端
cd frontend
npm run dev
```

### 2. 訪問簡易上傳頁面

在瀏覽器中打開：`http://localhost:5173/simple`

### 3. 填寫和上傳

1. 輸入**案件名稱**（例如：XX公司自動化專案）
2. 選擇**會議逐字稿**檔案（.txt, .doc, .docx）
3. （可選）選擇**工廠機器照片**（可多選）
4. 點擊「**開始處理並生成文件**」按鈕

### 4. 等待處理

系統會自動執行以下步驟：
- 建立案件
- 上傳檔案
- 提取需求（使用LLM）
- 產生報價方案
- 生成文件

處理過程中會顯示狀態訊息。

### 5. 下載文件

處理完成後，在頁面下方會顯示生成的文件卡片，點擊「下載」按鈕即可下載。

## 注意事項

1. **逐字稿格式**：建議使用純文字檔（.txt）或Word文件（.doc, .docx）
2. **圖片格式**：支援常見圖片格式（.jpg, .png, .gif等）
3. **處理時間**：根據逐字稿長度，處理可能需要1-3分鐘
4. **LLM配置**：確保在 `backend/.env` 中配置了正確的 LLM_API_KEY

## 文件說明

生成的文件包括：

- **需求規格書**：結構化的需求說明
- **需求報告表**：需求表格，包含證據片段
- **報價單**：三個自動化方案（P1/P2/P3）的報價

每個文件都有 DOCX 和 PDF 兩種格式。

## 故障排除

如果遇到問題：

1. **上傳失敗**：檢查檔案大小和格式
2. **提取失敗**：檢查 LLM API 配置
3. **文件未生成**：檢查 Celery Worker 是否運行
4. **下載失敗**：檢查後端日誌

## 完整功能

如果需要更多功能（編輯需求、調整報價等），可以訪問完整功能頁面：`http://localhost:5173`
